# Law-Judge 民商事司法分析引擎 (外商投资纠纷专精版)

基于多智能体协作 (MAS) 与 混合检索 (Hybrid RAG) 的法律决策支撑系统，专注于复杂民商事案件（特别是涉及外商投资）的事实锁定、压力质证与模拟法庭裁判。

---

## 核心功能矩阵

### 1. 事实查明与静态插槽 (Phase 1)
- **案由自动分类**：识别股权转让、公司决议、商事合同等 4 大类核心案由。
- **静态插槽锁定 (Static Slots)**：基于“请求权基础”理论，强制查明案件构成的必填要素（如支付凭证、工商登记等）。
- **对抗性事实质证**：模拟对抗视角，基于历史判例中的被告抗辩逻辑，对用户主张进行“压力测试”。

### 2. 模拟法庭与自动裁判 (Phase 2)
- **双子图流程编排**：由事实锁定（子图1）无缝切换至模拟辩论（子图2）。
- **多智能体博弈**：原告代理、被告代理、中立裁判官三个 Agent 进行 Agent-to-Agent 闭门辩论。
- **IRAC 推理模型**：遵循“争点(Issue)-规则(Rule)-适用(Application)-结论(Conclusion)”法律逻辑。
- **自动文书生成**：一键生成符合法院规范的民事裁判文书初稿。

### 3. 数据层与检索优化
- **混合检索 (Hybrid RAG)**：集成 **ChromaDB (向量检索)** 与 **Whoosh (全文关键词检索)**，解决法律名词的精准匹配。
- **异步文件入库**：支持 Word 文档解析，具备任务轮询与状态追踪。

## 部署与运行步骤

### 环境要求
- **Python**: 3.10+
- **Node.js**: v18+（及对应 NPM）

### 1. 一键部署 (Windows)
在项目根目录下双击运行：
```bash
setup.bat
```
该脚本将自动安装后端 Python 依赖及前端 NPM 依赖。

### 2. 参数配置
在项目根目录创建或编辑 `.env` 文件，确保包含以下配置：
```env
DEEPSEEK_API_KEY=您的API_KEY
SILICONFLOW_API_KEY=您的API_KEY
SEEKDB_PATH=./data/seekdb
```

### 3. 启动项目
- **启动后端**：运行 `run_backend.bat` (监听端口: 8001)
- **启动前端**：运行 `run_frontend.bat`

## 技术架构
- **Orchestration**: LangGraph
- **Backend API**: FastAPI + Uvicorn
- **Storage**: ChromaDB + Whoosh
- **Frontend**: React + Vite + Framer Motion
